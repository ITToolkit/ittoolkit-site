01 Introduction

This Privacy Policy describes how ITToolkit ("the app", "we", "us") handles information when you use the application on your Android device.

ITToolkit is a network diagnostics toolkit that provides tools such as DNS lookup, port scanning, traceroute, SSL certificate inspection, HTTP analysis, Wi‑Fi diagnostics, DNS propagation checks, speed tests, saved history, monitoring features, and export functions such as CSV and JSON where available.

ITToolkit is designed so that most diagnostic activity, preferences, saved items, and exported results remain on your device unless you choose to share them. Some limited data is also processed by Google services used by the app, including Google Play, Firebase Analytics, Firebase Crashlytics, and Firebase Firestore, as described in this policy.

02 Information the App May Access

Depending on the features you use, ITToolkit may access the following categories of information:

  • Network information — hostnames, domain names, IP addresses, URLs and port numbers entered by you when running diagnostic tools.
  • Wi‑Fi information — network names (SSIDs), access point identifiers (BSSIDs), signal strength, frequency, channel, connection state and nearby network scan results when using Wi‑Fi features.
  • Location-related access — location-related permissions used on-device where Android requires them for Wi‑Fi scanning and nearby network discovery.
  • App state and settings — user preferences, onboarding completion status, custom DNS settings, free usage counters, monitoring settings, and Pro upgrade status.
  • User‑entered data — saved hosts, bookmarks, lookup history, watchlist entries, and similar diagnostic items stored at your direction.
  • Crash and performance data — diagnostic information collected automatically by Firebase Crashlytics and Firebase Analytics to help monitor app stability and feature usage.
  • Device-linked usage sync data — where used by the app, a device identifier together with tool usage counts may be processed through Firebase Firestore to help prevent usage counters being reset after app data is cleared and to maintain app state across installs on the same device.

ITToolkit does not continuously track your location and does not store location history. Location-related access is used only where needed for Wi‑Fi-related features on Android.

03 Data Stored Locally on Your Device

The following data may be stored locally on your device to support app functionality:

  • Lookup history and tool results
  • Saved hosts, bookmarks and watchlist entries
  • App preferences and display settings
  • Free usage counters and Pro upgrade state
  • Notification and monitoring preferences
  • Files you export to your device, such as CSV or JSON files where those export options are available

This data remains on your device unless you clear app data, remove the relevant content, or uninstall the application. Exported files are stored on your device and are not sent to the developer unless you actively choose to share them using your device or another app.

Data processed by Google services such as Firebase Analytics, Firebase Crashlytics, Firebase Firestore and Google Play is retained according to those providers' own retention practices and policies. ITToolkit does not directly control the retention period of data processed by those third-party services.

Encrypted local storage

Some local app data, such as Pro status and usage-related settings, is stored using Android's encrypted preferences support to help protect it against unauthorised access on the device.

04 Permissions Used

Internet

Required to perform network diagnostic operations including DNS lookups, SSL certificate checks, HTTP inspection, port scanning, traceroute, speed tests, app updates from Google services, and other online diagnostics.

Network State & Wi‑Fi State

Used to detect connectivity and read Wi‑Fi connection details for diagnostic features.

Access Fine Location

On some Android versions, this permission is required for Wi‑Fi scanning and nearby network discovery. ITToolkit does not use this permission for continuous location tracking and does not store location history.

Nearby Wi‑Fi Devices (Android 13+)

Required on supported Android versions to read nearby Wi‑Fi network details during scanning. ITToolkit uses this permission only for Wi‑Fi-related features.

Post Notifications

Used only for optional alerts and reminders generated by features you enable, such as SSL certificate expiry alerts or network monitoring notifications.

Vibration

Used for haptic feedback and notification behaviour where supported by your device and Android version.

05 How Information Is Used

Information accessed by ITToolkit is used only for the following purposes:

  • Executing the network diagnostic tools you request
  • Displaying Wi‑Fi, network and diagnostic information
  • Saving your preferences, history, bookmarks, monitoring settings and other app state
  • Generating exports on your device in the formats supported by the relevant tool
  • Delivering optional alerts and reminders for features you enable
  • Processing in-app purchases and restoring prior purchases via Google Play
  • Presenting optional in-app review prompts via Google Play
  • Identifying and resolving crashes and performance issues via Firebase Crashlytics
  • Understanding aggregate and anonymised usage patterns via Firebase Analytics to guide future development
  • Maintaining tool usage counts or related app state via Firebase Firestore where device-linked usage sync is enabled in the app build

No information is used for advertising, data brokerage, or marketing profiling by the developer.

06 External Network Requests

As a network diagnostics application, ITToolkit makes outbound connections as a direct consequence of the tools and features you use. Depending on the feature, the following connections may be made:

  • DNS tools query the resolver specified by you, including custom resolvers where supported
  • HTTP Inspector connects to URLs you provide
  • SSL tools connect to the host or domain you specify to retrieve certificate data
  • Port Scanner and Traceroute connect to the hosts and IPs you enter
  • DNS propagation checks may query multiple public resolvers worldwide
  • Speed test functionality may connect to third-party measurement endpoints
  • Google Play handles billing, purchase restoration and review prompts

Connections initiated by ITToolkit use the network protocol required by the selected tool or test. Where applicable, secure HTTPS or TLS connections are used. Other tools may use DNS, TCP, SSL/TLS handshakes, ICMP-equivalent diagnostics, or user-specified protocols and ports as required by the test you run.

Firebase Analytics & Crashlytics

ITToolkit uses Firebase Analytics and Firebase Crashlytics, services operated by Google. These services may automatically process data such as:

  • Crash reports and stack traces when the app encounters an error
  • App stability and performance metrics
  • Usage events showing which app areas or tools are opened or used, without including the content of the hostnames, URLs, domains, or other diagnostic targets you manually enter

This data is used to improve app quality, reliability and usability. It is not used by the developer for advertising or sold by the developer to third parties.

Firebase Firestore usage sync

ITToolkit may use Firebase Firestore to store limited app state relating to usage control and restoration. Where this feature is used, the stored data may include a device-linked identifier and tool usage counts. This helps prevent free usage counters from being reset simply by clearing app data and helps preserve relevant app state on the same device.

This usage-sync data is not used to analyse the content of your diagnostic lookups. It is used only for app operation, entitlement handling, and usage-limit enforcement.

08 Security

The following measures are in place to help protect your data:

  • Some sensitive local app data is stored using Android encrypted preferences support
  • Outbound connections use the secure or diagnostic protocol required by the selected tool, including HTTPS or TLS where applicable
  • The developer does not operate a separate custom backend for storing your diagnostic targets or exported results

While reasonable safeguards are used, no system can be guaranteed completely secure. If you believe there is a security issue affecting ITToolkit, please contact support@ittoolkit.app.
